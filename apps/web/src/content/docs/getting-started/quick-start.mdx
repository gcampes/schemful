---
title: Quick Start
description: Define your first Contentful content model in TypeScript and push it to your space in under five minutes.
---

## 1. Initialize a project

```bash
schemful init
```

This creates:

- `schemas/` — where your content type definitions live
- `migrations/` — where generated migrations are stored
- `schemful.config.ts` — project configuration

## 2. Add your credentials

Create a `.env` file in the project root:

```bash title=".env"
CONTENTFUL_MANAGEMENT_TOKEN=CFPAT-xxxxxxxxxxxxx
CONTENTFUL_SPACE_ID=your_space_id
CONTENTFUL_ENVIRONMENT_ID=master
```

See the [Configuration](/getting-started/configuration) page for details on where to find these values.

## 3. Test the connection

```bash
schemful test
```

This verifies that Schemful can reach your Contentful space. You should see a success message with your space name and environment.

## 4. Define a content type

Create a schema file at `schemas/blogPost.ts`:

```ts title="schemas/blogPost.ts"
import { contentType, fields } from 'schemful';

export const blogPost = contentType('blogPost', {
  name: 'Blog Post',
  fields: {
    title: fields.symbol({
      name: 'Title',
      required: true,
    }),
    body: fields.richText({
      name: 'Body',
    }),
    publishedAt: fields.date({
      name: 'Published At',
    }),
  },
});
```

## 5. Push to Contentful

```bash
schemful push
```

Schemful diffs your local schemas against the remote content model, generates the necessary migration, and applies it to your Contentful environment.

## 6. Verify everything is in sync

```bash
schemful check
```

If your local schemas match the remote content model, you'll see:

```
✓ Everything is in sync.
```

That's it — your content model is now defined in code and deployed to Contentful.
